# Data Biz: Visual Design System

This document defines the visual "vibe" and design system for Data Biz. It is used to ensure consistency across all digital touchpoints.

## Visual Design System (The "Vibe")

### Color Palette
- **Background:** Warm off-white or soft beige (e.g., `#FDFCFB` or `hsl(30, 20%, 98%)`).
- **Text:** Deep charcoal or soft black (e.g., `#1A1A1A` or `hsl(0, 0%, 10%)`).
- **Accent:** One restrained, professional teal (e.g., `#008080` or `hsl(180, 100%, 25%)`). Use sparingly for CTAs or key highlights.
- **Subtle Layering:** Use light grey borders (`#E5E5E5`) or soft shadows for depth.

### Typography
- **Headings:** Premium editorial serif (e.g., "Playfair Display", "Outfit", or a system serif).
- **Body:** Clean, highly readable sans-serif (e.g., "Inter", "Roboto", or system sans).
- **Alignment:** Left-aligned by default. Avoid centered text for long blocks to maintain an editorial feel.
- **Hierarchy:** Strong contrast between heading sizes and body text.

### Layout & Composition
- **Rhythm:** Asymmetric but clean. Avoid the "centered everything" SaaS landing page pattern.
- **Whitespace:** Generous margins and padding. Let the content breathe.
- **Grids:** Use clean, structured grids for lists and cards, but break the symmetry occasionally for a bespoke feel.

### Motion & Interaction
- **Minimalist:** No dramatic parallax or bouncy animations.
- **Crisp:** Hover states should be intentional and subtle (e.g., slight color shift or underline).
- **Soft Reveals:** Use gentle fades or vertical slides for page entry.
